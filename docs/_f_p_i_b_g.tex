All classes will have the methods even if they are not used. The following is an example class. It contains the constructor in which we save the object name. It has a create function that assigns the base object to the member self.\+bobj. It assigned the config and log file objects to short names to make the code more compact. It also assignes the error code level to {\ttfamily python self.\+dlvl} of this class which insures that we have a unique error id for any errors See Section \doxysectlink{_f_p_i_b_g_section3}{Logging}{1}.

Also notice that assignment of all configuration items needed for the class are contained in the Create() function and multiple items are arranged into a try block. This is to insure that all configuration items you need exist before the class is used.

\hypertarget{_f_p_i_b_g_section3}{}\doxysection{\texorpdfstring{Logging}{Logging}}\label{_f_p_i_b_g_section3}
We log all major events for success and failure with a debug verbosity level. The level is from 0 to 5, 5 being the most verbose. \hypertarget{_f_p_i_b_g_subsec1}{}\doxysubsection{\texorpdfstring{Logging succesfull function calls.}{Logging succesfull function calls.}}\label{_f_p_i_b_g_subsec1}
Verbose level of 0 is for any successful opening of a file or communications port, testing a directory for existence, or any other major function call. Even though these are succesfull we need to see a sort of logging stack when we review the log file. It is really important to log as much as you can.

Also, log reads and writes but set the verbosity level to 5 since this will usually flood the log file and needs to be used in rare cases.

\hypertarget{_f_p_i_b_g_section4}{}\doxysection{\texorpdfstring{Exercising and Unit testing}{Exercising and Unit testing}}\label{_f_p_i_b_g_section4}
There is a difference between Exercising and Unit Testing.\hypertarget{_f_p_i_b_g_subsec5}{}\doxysubsection{\texorpdfstring{Exercising}{Exercising}}\label{_f_p_i_b_g_subsec5}
Exercising code means writing code that performs a specific function. This code resides in the {\ttfamily test} directory. There are three reasons for this.


\begin{DoxyItemize}
\item To push developers to write code modualarly, in small bites
\item To have example code avaiable for new developers who may need to extend what the original developers have done.
\item So that the code reviewer can review the developer tasks more easily.
\end{DoxyItemize}

When you become project mangers you will understand and appreciate the importants of this approach.\hypertarget{_f_p_i_b_g_subsec6}{}\doxysubsection{\texorpdfstring{Unit Testing with Pytest}{Unit Testing with Pytest}}\label{_f_p_i_b_g_subsec6}
